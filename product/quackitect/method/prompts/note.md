# note — capture an idea (do not process it)

Frictionless. Write exactly one note file to .quack/notes/inbox/ and stop.
- Record the idea in the user's own words.
- ALWAYS ask the user for the **origin** (where/how the idea arose), in their own words —
  never infer it. (The `quack note` CLI lane stamps `origin: commandline` instead.)
- If the note is too thin to be useful later, ask at most ONE batched clarifying question.
- Do not edit spec, checks, or the backlog. Confirm tersely and stop.
A note is never a check until `engage start` bakes it. Later sub-op: `comment`.
